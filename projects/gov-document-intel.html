Overview

The 20-Minute Bottleneck

Bangladesh government ministries process 10+ million paper documents annually (birth certificates, trade licenses, land deeds) through manual data entry by clerks. This process is slow (20 minutes per document), error-prone (8-12% transcription error rate), and vulnerable to corruption.

Digital Bangladesh Vision 2041 demands automation, but off-the-shelf OCR fails on Bangladeshi forms: complex layouts, mixed Bengali/English text, handwritten sections, and low-quality photocopies.

The RAG + VLM Pipeline

We built a pipeline combining three models: (1) **LayoutLMv3** (175M parameters) fine-tuned on 1,000 examples across 10 government form types, (2) **Bangla-BERT** (BUET's 110M parameter model) for language-specific understanding, (3) **Retrieval-Augmented Generation (RAG)** system querying Bangladesh legal code to flag compliance issues.

The workflow: 1. Image preprocessing (Deskewing, Binarization) 2. LayoutLMv3 extraction (Predicts labels like 'Name', 'Amount' + bounding boxes) 3. Bangla-BERT validation (Checks Bengali text fields for correctness) 4. Structured JSON output 5. RAG compliance check (FAISS vector database of laws + Llama 3.1 70B for analysis) 6. Human-in-loop review for flagged edge cases (10-15% of volume).

Total pipeline latency: 5-8 seconds per document (vs. 20 minutes manual).
